# 隧道施工场景纯视觉避障 方案分析

[官方资料](https://opencv.org/) · [Ultralytics YOLOv8](https://docs.ultralytics.com/)

## 场景定义

### 环境结构

| 要素 | 位置 | 视觉特征 | 约束 |
|------|------|---------|------|
| 隧道墙壁 | 两侧最外 | 混凝土/瓷砖纹理，纵向延伸 | 绝对不可跨越 |
| 凸起隔离带 | 路两侧（靠隧道壁） | 纵向凸起边缘，亮度突变 | 不可跨越 |
| 中央隔离沟 | 路面正中间 | 纵向暗色凹槽/线条 | **绝对不能跨越** |
| 半幅车道 | 隔离沟与隔离带之间 | 平坦水泥/沥青路面 | 可行驶区域 |
| 小车定位 | 隔离沟一侧（半幅路内） | — | 不得进入另一侧 |

### 车辆平台

| 参数 | 特征 | 对算法的影响 |
|------|------|-------------|
| 车轮 | 前后四轮等径 | 阿克曼转向，需考虑内轮差 |
| 底盘 | 离地间隙较大 | 可跨越碎石/细管线(<5cm)，不可压钢筋/大石块(>10cm) |
| 车身宽度 | ~1.5-2.0m (估算) | 半幅路绕行空间有限(0.9-1.9m) |
| 行驶速度 | 10-30 km/h | 低速场景，但制动距离仍不可忽略 |
| 摄像头 | 单目，车头安装 | 竖屏 720×1280 (当前) |

## 感知层挑战（33项）

### 极端光照

| # | 挑战 | 失效机制 | 影响模块 | 严重度 |
|---|------|---------|---------|--------|
| 1 | 大灯/电弧过曝 | CMOS 像素饱和(RGB=255,255,255)，纹理丢失，光斑内障碍物不可见；CCD blooming 产生垂直 smear | Canny、YOLOv8、HSV | 🔴 致命 |
| 2 | 大灯开关瞬态 | 0.1→10000 lux 阶跃跳变，AE 需 5-15 帧调整，期间全白/全黑，调整过程过冲-回调-振荡 | 全部模块 | 🔴 致命 |
| 3 | 隧道出入口切换 | 自然光过渡比人工光快数倍，AE 窗口 10-20 帧，盲区行驶距离 2m+(@20km/h) | 全部模块 | 🔴 致命 |
| 4 | 施工灯光明暗条纹 | 间隔布置的临时灯→AE 帧间来回调，一帧过曝一帧欠曝 | Canny、HSV | 🟠 严重 |
| 5 | 光幕散射 | 粉尘+灯光散射→全局对比度崩塌(veiling glare)，CLAHE 在此场景无效 | CLAHE、Canny | 🔴 致命 |

### 传感器噪声与退化

| # | 挑战 | 失效机制 | 影响模块 | 严重度 |
|---|------|---------|---------|--------|
| 6 | 低照度 CMOS 噪声 | 高 ISO 下散粒噪声+读出噪声+热噪声，暗区像素跳动 ±20 级→Canny 满屏假边缘→findContours 连片 | Canny、障碍检测 | 🔴 致命 |
| 7 | 镜头污染(泥/尘) | 泥斑=永久遮挡，每帧位置固定→时域滤波无效→持续误报；粉尘积累=渐进退化 | 全部模块 | 🔴 致命 |
| 8 | 镜头水滴/冷凝 | 水滴=微型透镜→局部折射/模糊；冷凝=内表面起雾无法擦拭；振动时水滴晃动→假运动信号 | Canny、YOLOv8 | 🟠 严重 |
| 9 | 镜头油污干涉 | 油膜薄层干涉→彩色条纹→HSV 地面分割误判为非地面 | HSV | 🟡 重要 |
| 10 | 热噪声累积 | 芯片升温→暗电流增加→噪点随时间变多，长期运行后算法突然失效 | Canny | 🟡 重要 |

| 38 | 动态速度上限 | 环境质量→速度映射: 照明差→20km/h, 湿滑→15km/h, 粉尘→10km/h, AE 调整中→5km/h | 🟠 严重 |

### 控制与系统安全

| # | 问题 | 核心难点 | 严重度 |
|---|------|---------|--------|
| 39 | 转弯+制动耦合 | 转弯中急刹→离心力+减速度>摩擦圆→侧滑；绕行时须限制最大速度 | 🟠 严重 |
| 40 | 死锁检测与脱困 | 前进/后退/左右全堵→持续 STOP 不是解决方案→需声光报警/请求人工接管 | 🟡 重要 |
| 41 | 多障碍物时空推理 | 同时 N 个威胁+各自轨迹→需要预测未来状态做全局最优，非单帧贪心 | 🟠 严重 |
| 42 | 自身货物安全 | 急刹→松散货物前冲/倾覆；需比纯安全策略更温和(减速度上限 -3m/s²) | 🟠 严重 |
| 43 | 人机权限分配 | 常规: 算法建议+操作员确认；紧急: 算法直接执行；操作员超控有 1s 延迟 | 🟠 严重 |
| 44 | 能耗与磨损优化 | 减少不必要急刹，平衡安全与运维成本 | 🟢 需考虑 |
| 45 | Fail-Safe 链路 | 任意模块失效→自动向安全状态退化: YOLOv8 崩→限速, 摄像头断→停车, 断电→物理抱死 | 🔴 致命 |

### 安全设计原则

```text
安全偏向(Safety Bias):
  不确定时偏向保守 — 类别不确定 → 按最危险处理
  距离不确定 → 按最近距离处理
  路面不确定 → 按最低摩擦系数处理

失效安全(Fail-Safe):
  模块失效 → 系统向安全状态退化，不可继续正常运行

最小惊讶(Least Surprise):
  同一场景 → 同一决策
  减速 → 平缓(不突然急刹)
  绕行 → 提前信号灯/鸣笛
```

## 路径规划层挑战（10项）

### 为什么反应式策略不够

```text
当前决策: 中心阻塞 → 左右选更畅通一侧 → 单步绕行

失效场景:
  连续交替障碍 → 车在车道内蛇形摆动
  非凸障碍布局 → 需多步动作序列，单帧评分无法规划
  通道宽度 < 安全阈值 → 算法判断"可行"但实际无法通过
```

### 可通行性判断

| # | 问题 | 核心难点 | 严重度 |
|---|------|---------|--------|
| 46 | 多步路径规划 | 非凸障碍布局下反应式策略失效，需规划一串动作序列 | 🔴 致命 |
| 47 | 可通行性精确判断 | 间隙 > 车宽 是必要条件非充分条件；还需安全余量、接近角度、内轮差、车身姿态 | 🔴 致命 |
| 48 | 内轮差避碰 | 转弯时内侧后轮轨迹半径最小(比外前轮小 0.5-0.8m)，易刮擦障碍物/隔离带 | 🟠 严重 |
| 49 | 像素→度量空间标定 | 路径规划需米级精度，单目摄像头缺乏尺度信息 | 🔴 致命 |
| 50 | 规划失败降级 | DWA 无解→回退反应式→反应式失败→减速停车→人工接管，需平滑过渡 | 🟠 严重 |
| 51 | 动态障碍物截断规划 | 规划路径被新出现的动态障碍物拦截→需实时碰撞检查+快速重规划(33ms内) | 🟠 严重 |
| 52 | 局部规划近视 | DWA 只看到~10m，12m 外的死胡同检测不到→需要粗粒度远距离层 | 🟡 重要 |
| 53 | 规划/控制频率解耦 | 每帧重规划→方向盘高频颤动；需规划 10Hz + 控制 30Hz + 平滑插值 | 🟡 重要 |
| 54 | 路径跟踪精度 | 转向延迟、路面不平、轮胎侧偏→实际偏离规划路径→需实时反馈修正(pure pursuit) | 🟠 严重 |
| 55 | 倒车/脱困规划 | 前进无路时需倒车找新路径；倒车有额外盲区和运动学约束 | 🟡 重要 |

### 推荐技术路线

```text
局部规划: DWA (Dynamic Window Approach)
  理由: 低速场景适配、运动学约束自然纳入、计算快(5-20ms)、输出平滑

备选: TEB (Timed Elastic Band)
  理由: 时间最优 + 运动学，精度更高但耗时略大(10-50ms)

降级链: DWA → 反应式避障 → 减速停车 → 紧急制动 → 人工接管
```

## 技术栈建议

```text
输入层:
  单目摄像头 (当前)
  可选: 红外热成像 (夜间行人检测)
  可选: IMU (振动补偿/运动估计)

感知层:
  路面分析:  纹理一致性 + 单目深度估计(Depth Anything v2 small)
  结构检测:  消失点约束 + 纵向边缘连续性 → 隔离带/沟定位
  动态障碍:  YOLOv8 + ByteTrack (检测+跟踪)
  光照鲁棒:  CLAHE增强 + 过曝区域 mask + 时域降噪(3DNR)

决策层:
  可通行性:  像素→度量空间标定 → BEV 占据栅格
  路径规划:  DWA (10Hz 局部规划) + 粗粒度远距离层
  安全降级:  规划 → 反应式 → 减速 → 停车 → 紧急制动 → 接管

控制层:
  路径跟踪:  Pure Pursuit / Stanley Controller
  速度控制:  动态上限(环境质量 → 速度映射)
  Fail-Safe: 任意模块失效→向安全状态退化(含断电物理抱死)
```

## 注意事项

- 所有 55 项挑战均为定性分析，具体参数（HSV 阈值、Canny 阈值、障碍面积范围）需在真实场景视频上迭代调参
- 像素→度量空间标定是感知到规划的关键桥梁，需优先解决
- 时域信息（多帧关联）是当前算法最大缺失，涉及飞溅物过滤、噪声抑制、AE 切换检测等多项挑战
- 镜头污染是最容易被低估的问题，建议硬件侧加装防护罩+定期清洁方案
- 极端光照(AE 切换、大灯开关)的盲区窗口无法通过纯算法消除，需靠降速兜底
