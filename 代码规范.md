# 团队代码规范

## 0\. 总原则

1. **简洁** —— 能用函数就不写类，能改全局就不上 argparse

2. **层级清晰** —— 一个文件一个职责；底层 wrapper 与业务逻辑必须分开文件

3. **可读** —— 板块分隔 \+ 命名规范 \+ 必要时才注释（英文）

4. **易切换** —— 切换参数/方法靠改全局或注释掉旧调用，不要写 if/else 分支或命令行参数

5. **检查错别字\!\!**

## Python

### 1\.1 命名

### 1\.2 类 vs 函数

**只在以下情况写类**：

- 需要长期持有状态（连接 handle、缓存、线程）

- 底层硬件 / 协议包装（机器人、相机、串口、CAN）

- 同一接口要 real / mock 两种实现

**其余一律用模块级函数 \+ 模块级状态**。

**不需写 abstract 基类（import abc）**

### 1\.3 参数 / 方法切换 

不用 argparse切换

**\(a\) 调参 → 顶部全局变量**

```Plain Text
==== TUNABLE PARAMETERS ====
YOLO_CONF_THRESHOLD = 0.5
VISIBLE_COUNT_THRESHOLD = 5
STOP_REGION = {'x_min': 0.3, 'x_max': 0.7, ...}
```

**\(b\) 换方法 → main 里注释切换**

```Plain Text
if **name** == "**main**":
    robot = DucoRobot()
    # robot = MockRobot()

    robot.move_joints(poses[3])
    # robot.move_through_poses(poses)
```

不要写 `--mode` 参数，不要写 `if config == "a": ...`。

### 1\.4 文件结构

```Python
#!/usr/bin/env python3
"""One-line module purpose."""
import os
import sys
...

==== TUNABLE PARAMETERS ====
CONST_A = ...

==== STATE ====              # only if module-level state exists
state_x = None

==== CORE ====
def public_api(...): ...

==== HELPERS ====
def _helper(...): ...

==== TEST ====
if **name** == "**main**":
    ...
板块用 # ==== NAME ==== 分隔，全大写。
```

### 1\.5 `main` 作为测试入口

- 直接 `python xxx.py` 等于跑测试

- 其他文件用 `from xxx import ClassA, func_b` 调用

- 多个测试用例 → 注释切换，不要新增参数

### 1\.6 注释

- 英文，简短

- 公共函数 / 类一行 docstring

- 不解释 "what"（代码自己说），只解释 "why" 或不明显的常量来源

- 不要逐行注释 `# loop over frames`

### 1\.7 打印日志

统一前缀，便于过滤：

```Python
print(f"[OK] Connected to {ip}")
print(f"[WARN] Library not available, using mock")
print(f"[ERR] ...")
print(f"[TARGET] Model loaded")     # 模块名 tag
```

### 1\.8 不要做的事

- 不写空类、空 `init`

- 不写 getter/setter（Python 直接属性访问）

- 不留大段被注释掉的旧代码（用 git）

- 不在生产代码里留 `print("here")` 调试语句

### 1\.9 示例

```Python
#!/usr/bin/env python3
"""
DucoCobot Driver
"""
import os
import sys
import time
import threading
import numpy as np
import math
from scipy.spatial.transform import Rotation as R
script_dir = os.path.dirname(os.path.abspath(__file__))
sys.path.insert(0, script_dir)

from duco_robot.DucoCobot import DucoCobot
ROBOT_AVAILABLE = True

class DucoRobot:
    """Simple wrapper for DucoCobot"""
    
    def __init__(self, ip='192.168.3.10', port=7003):
        self.ip = ip
        self.port = port
        self.robot = DucoCobot(ip, port)
        
        self.teach_mode = False
    
    # ===== POSE =====
    def get_pose(self):
        """Get current TCP pose [x, y, z, rx, ry, rz]"""
        return self.robot.get_tcp_pose()
    
    def stop(self):
        """Stop all motion"""
        self.robot.speed_stop(False)
    
    # ===== CLEANUP =====
    def close(self):
        """Close connection"""
        if self.teach_mode:
            self.disable_teach_mode()
        self.stop()
        if ROBOT_AVAILABLE:
            self.robot.close()
        print("[OK] Robot disconnected")

# ============================================================
# TEST
# ============================================================

if __name__ == "__main__":
    print("\n" + "="*60)
    print(" Clean DucoRobot Test")
    print("="*60)
    
    robot = DucoRobot()

    print("\nCurrent joints:", robot.get_joints())
    print("\nCurrent pose:", robot.get_pose())

```

### 2\.2 ROS Node 模板（单文件单类）

```C++
#include <ros/ros.h>
// ... other headers

// One-line node purpose.

class XxxNode {
public:
    XxxNode() {
        sub_ = nh_.subscribe("in_topic", 10, &XxxNode::callback, this);

private:

    ros::Subscriber sub_;
    ros::Publisher  pub_;
    ros::Timer      timer_;

    float cached_value_ = 0.0f;

    void callback(const MsgType::ConstPtr& msg) { ... }
    void timerCb(const ros::TimerEvent&)        { ... }
};

int main(int argc, char** argv) {
    ros::init(argc, argv, "xxx_node");
    XxxNode node;
    ros::spin();

```

### 2\.3 头文件

- 单文件 node 不需要 `.h`

- 仅当代码被多个 `.cpp` 复用时才建头文件

- 头文件路径：`include/<package_name>/xxx.h`

- 用 `#pragma once`，不用传统 include guard

### 2\.4 注释

- 英文

- 文件顶部 1–3 行说明用途（参考 `can_reader_node.cpp`）

- 魔法常数 / 复杂公式旁标注来源

- 未实现部分用 `// TODO: ...`

### 2\.5 风格细节

- 缩进 4 空格

- 大括号 `{` 不换行（K\&R 风格）

- 优先 `float` / `double` 明确选择，不要 `auto` 用在原生类型上

- 不写 `using namespace std;` 在头文件

## ROS 包

### 3\.1 包命名 `<project>_<purpose>`

### 3\.2 目录约定

```Python
<package>/
├── CMakeLists.txt
├── package.xml
├── include/<package>/    # C++ 头文件
├── src/                  # C++ 源码
├── scripts/              # Python 脚本（chmod +x，#! shebang 必须）
├── launch/               # *.launch
├── config/               # *.yaml 参数
├── rviz/                 # *.rviz
├── msg/                  # 仅 *_msgs 包
└── urdf/                 # 仅 *_description 包
```

### 3\.3 一个包做一件事

- `*_msgs`：只放 `.msg` / `.srv` / `.action`，不放代码

- `*_description`：只放 URDF / mesh / 静态资源

- 控制 / 感知 / 底盘节点分到独立包

- **跨包通信只走 topic / service / action，禁止跨包 import 内部模块**

### 3\.4 消息命名

- 文件：`PascalCase.msg`

- 字段：`snake_case`

- 配对：`XxxCmd.msg` 配 `XxxStatus.msg`

### 3\.5 Topic 命名 —— 分层带命名空间

```Bash
/vehicle/status/wheel
/vehicle/status/lift
/vehicle/status/support
/vehicle/status/battery_pct
/vehicle/cmd/...
/perception/line
/perception/heading
/robot/arm/...
```

不要顶层裸名（`/wheel`, `/speed`）。

### 3\.6 Launch

- 入口：`<package>_system.launch` 启全系统

- 子集：`<package>_<subset>.launch`（如 `avs_perception_only.launch`）

- 参数抽到 `config/*.yaml`，launch 里 `<rosparam file="..." />` 加载

- 不在 launch 里硬编码 IP / 端口 / 阈值

## Git Commit

- 英文，前缀分类：

    - `[feat]` 新功能

    - `[fix]` 修 bug

    - `[refactor]` 重构（不改行为）

    - `[docs]` 文档

    - `[chore]` 构建 / 工具链

- 单次 commit 单一目的，不混提交

- 例：`[feat] avs_perception: add line detector node`
